How Pre-Market Arb Works

NSE and BSE run separate pre-open sessions (9:00–9:08 AM). During this time, buy/sell orders are collected and an IEP (Indicative Equilibrium Price) is computed independently on each exchange. Because they're separate auctions, the same stock can have meaningfully different IEPs — that's the arb opportunity.

By 9:07–9:08 AM these prices settle. You want to buy on the cheaper exchange and sell on the dearer one right as the market opens at 9:15 AM (or as close to it as possible). The spread tends to close quickly after open.

EXAMPLE — SHADOWFAX
BSE IEP: ₹117.25  |  NSE IEP: ₹128.00
Spread: ₹10.75  |  Arb %: 9.17%
Action: BUY BSE + SELL NSE at open
Data Source Options
Option 1 — Kite Connect API (Best): Gives live IEP data during pre-open via GET /quote?i=NSE:SYM&i=BSE:SYM. The last_price field reflects IEP during pre-open. Requires ₹2000/mo Kite Connect subscription.

Option 2 — Local Backend (Free): Run arb_backend.py locally. It fetches NSE + BSE pre-open APIs (free, no key) and serves them via http://localhost:5050/quote. CORS is handled automatically — no proxy needed.

Option 3 — Simulation: Demo mode. Use to test the UI without a live data connection.
Backend Endpoints Reference
# Start the backend
python arb_backend.py

# Endpoints
GET http://localhost:5050/health
    → {"status":"ok","mode":"nse+bse","nse_symbols_cached":150,...}

GET http://localhost:5050/quote?sym=RELIANCE&sym=SWIGGY
    → {"RELIANCE":{"nse":1240.00,"bse":1238.50,"arb_pct":0.1210,...},...}

GET http://localhost:5050/preopen/all?threshold=3
    → {"count":4,"opportunities":[{"symbol":"SHADOWFAX",...},...]}

# Config flags in arb_backend.py
USE_KITE = False          # set True + fill keys for Kite
CACHE_TTL = 8             # seconds between NSE/BSE re-fetches
